## Commands

```bash
npm install       # Install dependencies
npm run dev       # Start dev server at http://localhost:3000 (opens browser automatically)
npm run build     # Build to ./build directory
```

There is no test runner or TypeScript type-check script configured. `@vitejs/plugin-react-swc` transpiles TypeScript but does not perform type checking — `tsc` is not in devDependencies.

## Architecture

This is a single-page React + TypeScript app built with Vite. All game logic lives in `src/App.tsx` and all database/auth logic lives in `src/db.ts` — there is no routing, no global state library, and no backend.

**View flow:** App starts on an `'auth'` view (sign in / sign up / play as guest). On success it transitions to `'game'`. Sign-out returns to `'auth'`. The `view` state (`'auth' | 'game'`) drives which JSX branch renders.

**Game flow:** When `view` becomes `'game'`, `initializeGame()` runs via `useEffect` and randomly assigns a treasure to one of three `Box` objects. Clicking a closed box calls `openBox()`, which updates that box's `isOpen` flag, adjusts the score (+100 for treasure, -50 for skeleton), and checks for end-of-game (treasure found or all boxes opened). The "Play Again" button calls `initializeGame()` directly. On game end, the score is saved via `saveScore()` if a user is signed in (guarded by a `scoreSavedRef` to prevent double-saves).

**Key state shape** (`src/App.tsx`):
```ts
interface Box { id: number; isOpen: boolean; hasTreasure: boolean; }
interface User { id: number; username: string; }
interface ScoreRecord { score: number; created_at: string; }
type View = 'auth' | 'game';
// useState: view, dbReady, dbError, currentUser, isGuest, boxes[], score, gameEnded, scoreHistory
```

**Database layer** (`src/db.ts`): Uses `sql.js` (SQLite compiled to WASM) running entirely in the browser. The SQLite database is serialized and persisted to IndexedDB (`TreasureGameDB`) so data survives page reloads. `initDB()` loads the WASM binary from `/sql-wasm-browser.wasm` (in `public/`), restores a saved DB from IndexedDB or creates a fresh one, and creates the `users` and `scores` tables. Passwords are hashed with `crypto.subtle.digest('SHA-256')` (browser Web Crypto API — no bcrypt). Exported functions: `initDB`, `signUp`, `signIn`, `saveScore`, `getScores`.

**Assets:**
- Images: `src/assets/` — `treasure_closed.png`, `treasure_opened.png`, `treasure_opened_skeleton.png`, `key.png`
- Audio: `src/audios/` — `chest_open.mp3`, `chest_open_with_evil_laugh.mp3`
- WASM: `public/sql-wasm-browser.wasm` and `public/sql-wasm.wasm` — required by sql.js; do not remove
- Both images and audio are imported directly in `App.tsx` and resolved by Vite

**Animations:** Uses the `motion` package (the successor to Framer Motion), imported as `from 'motion/react'` — not `framer-motion`.

**UI components:** `src/components/ui/` is a full shadcn/ui component library (Radix UI primitives + Tailwind). Only `Button` is currently used by the game. `src/components/ui/utils.ts` exports the `cn()` helper (clsx + tailwind-merge) used throughout the ui components. `src/components/figma/ImageWithFallback.tsx` is a drop-in `<img>` replacement that shows a placeholder SVG on load error.

**Styling:** Tailwind CSS utility classes inline in JSX. Global base styles in `src/index.css` (large file — generated). `src/styles/globals.css` exists but is not imported anywhere.

**Path alias:** `@` resolves to `src/` (configured in `vite.config.ts`).

**Vite aliases:** `vite.config.ts` contains versioned package aliases (e.g. `'vaul@1.1.2': 'vaul'`) generated by Figma to resolve version-pinned imports from shadcn/ui components. Do not remove them.
